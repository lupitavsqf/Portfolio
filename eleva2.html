Ongoing research

A middleware layer for working memory between human interfaces and LLMs

This research explores a UI–backend orchestration layer that externalizes “working memory” for LLM-assisted work. The system mediates context, intent, and state across interactions, allowing users to reason, revisit, and intervene in how memory is constructed, persisted, and retrieved—without collapsing cognition into a single prompt or session.

System articulation

Externalizing working memory as an interfaceable system

The system models working memory as a persistent, inspectable layer that sits between user intent and model execution. Rather than relying on ephemeral prompts, it enables structured memory objects, selective recall, and explicit user intervention in how context is assembled.

This approach reframes LLM interaction as an ongoing cognitive process rather than a sequence of isolated requests. By treating memory as infrastructure, the interface supports revision, continuity, and reflective work over time.

How can a chat-based interface support extended cognition?

This question frames the research problem at the core of the project. Rather than treating chat as a conversational endpoint, the work explores how dialogic interfaces can function as durable cognitive scaffolds—supporting memory, revision, and reflective reasoning across time.

My proposal

A unified layer for LLM-assisted knowledge work

The proposal introduces a middleware layer that externalizes working memory between users and LLMs, enabling persistent context, selective recall, and explicit intervention. The design ensures transparency, cognitive accessibility, and extensibility for multiple workflows.

Interaction flows and feedback loops

The system leverages chat interfaces, structured memory objects, and visualization of memory traces to create a reflective and iterative workflow. Users can inspect, revise, and replay contextual states seamlessly.
